DNS Tunneling: Conceptos y Prevención


¿Qué es DNS Tunneling?

El DNS tunneling es una técnica utilizada por ciberdelincuentes para ocultar el tráfico malicioso en solicitudes DNS legítimas. Esto permite eludir las restricciones de seguridad y comunicarse con servidores remotos de comando y control (C&C) de manera encubierta.


Escenario de Ataque

Imaginemos una situación en la que una empresa, E-Corp, es víctima de un ataque de DNS tunneling. El atacante logra infiltrarse en la red corporativa y utiliza el DNS tunneling para comunicarse con servidores C&C externos y exfiltrar datos confidenciales.



Cómo el Atacante Realizó el DNS Tunneling

El atacante pudo haber llevado a cabo el ataque de DNS tunneling de la siguiente manera:

  1. Obtuvo acceso a la red corporativa de EjemploCorp mediante técnicas de intrusión.
  2. Configuró un servidor DNS malicioso dentro de la red de la empresa.
  3. Creó solicitudes DNS maliciosas que contenían datos encriptados y las envió al servidor DNS malicioso.
  4. El servidor DNS malicioso interpretó las solicitudes y desencriptó los datos, permitiendo la comunicación con los servidores C&C.
  5. El atacante pudo exfiltrar datos y controlar las operaciones maliciosas a través de esta comunicación encubierta.

Detección de Ataques de DNS Tunneling

El DNS tunneling implica el abuso del protocolo DNS subyacente. En lugar de utilizar solicitudes y respuestas de DNS para realizar búsquedas legítimas de direcciones IP, el malware lo utiliza para implementar un canal de comando y control con su controlador.

La flexibilidad de DNS lo convierte en una buena opción para la exfiltración de datos; sin embargo, tiene sus límites. Algunos indicadores de DNS tunneling en una red pueden incluir:

  • Solicitudes de Dominios Inusuales: El malware de DNS tunneling codifica datos dentro de un nombre de dominio solicitado (como DATOS_AQUÍ.baddomain.com). La inspección de los nombres de dominio solicitados en las solicitudes de DNS puede permitir a una organización diferenciar el tráfico legítimo de los intentos de DNS tunneling.
  • Solicitudes de Dominios Inusuales: El DNS tunneling solo funciona si el atacante es dueño del dominio de destino para que las solicitudes de DNS vayan a su servidor DNS. Si una organización experimenta un aumento repentino en las solicitudes de un dominio inusual, puede indicar DNS tunneling, especialmente si ese dominio se creó recientemente.
  • Alto Volumen de Tráfico DNS: El nombre de dominio en una solicitud de DNS tiene un tamaño máximo (253 caracteres). Esto significa que un atacante probablemente necesitará una gran cantidad de solicitudes de DNS maliciosas para realizar la exfiltración de datos o implementar un protocolo de comando y control altamente interactivo. El aumento resultante en el tráfico de DNS puede ser un indicador de DNS tunneling.

Todos estos factores pueden ser benignos por separado. Sin embargo, si una organización experimenta varias o todas estas anomalías, puede ser una indicación de que el malware de DNS tunneling está presente y activo en la red.

Cómo Protegerse contra el DNS Tunneling

Protegerse contra el DNS tunneling requiere un sistema avanzado de prevención de amenazas en la red capaz de detectar y bloquear esta tentativa de exfiltración de datos. Dicho sistema debe realizar inspección del tráfico de red y tener acceso a una inteligencia de amenazas sólida para respaldar la identificación del tráfico dirigido hacia dominios maliciosos y contenido malicioso que pueda estar incrustado en el tráfico de DNS.
